# Networking Labs

## Overview

This repository contains hands-on networking labs focused on core concepts used in enterprise and service provider networks. The labs are completed using **Cisco Packet Tracer** and are documented in a step-by-step format similar to a real-world troubleshooting or engineering ticket.

Each lab report includes:

* Network topology overview
* Device configuration steps
* Command-line interface (CLI) commands used
* Verification and troubleshooting procedures
* Screenshots demonstrating configuration and results

The goal of this repository is to document practical networking experience while reinforcing troubleshooting methodology and network engineering fundamentals.

---

## Repository Structure

Each lab is organized into its own directory and includes a detailed README explaining the configuration process and results.

Example structure:

```
Networking-Labs
│
├── Lab01-VLAN-Router-on-a-Stick
│   └── README.md
│
├── Lab02-Network-Troubleshooting
│
├── Lab03-Spanning-Tree-Analysis
│
├── Lab04-Routing-Fundamentals
│
└── Screenshots
```

The **Screenshots** directory stores supporting images used throughout the lab reports.

---

## Topics Covered

The labs in this repository focus on foundational networking skills used by network administrators, NOC engineers, and cybersecurity professionals.

Topics include:

* VLAN configuration
* Inter-VLAN routing
* Router-on-a-Stick architecture
* Layer 2 switching concepts
* Trunk ports and 802.1Q tagging
* Basic router configuration
* Network troubleshooting methodology
* CLI verification commands

Additional labs may explore:

* Spanning Tree Protocol (STP)
* Network segmentation
* Routing concepts
* Network failure scenarios and troubleshooting

---

## Tools Used

The following tools were used to build and test these networking environments:

* **Cisco Packet Tracer**
* **Cisco IOS CLI**
* Network troubleshooting commands such as:

```
show vlan brief
show interfaces trunk
show running-config
show ip interface brief
```
